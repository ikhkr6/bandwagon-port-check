## 搬瓦工端口测试之前，先搞清楚一件事

**端口测试**（Port Test）的核心是判断：你的搬瓦工VPS的某个端口，在国内是否能正常连通。

这和IP测试不是一回事。IP测试是看整个服务器的IP地址有没有被封，端口测试是看某个具体端口（比如SSH的22端口，或者你自己改过的端口）有没有被单独封掉。

为什么要区分这两个？因为现在的封锁机制已经很精准了——有时候IP本身是通的，ping也有响应，但SSH连接就是进不去，原因就是那个SSH端口被封掉了，IP本身没问题。光靠ping根本发现不了这种情况。

所以正确的排查顺序是：先测IP → 再测端口 → 根据结果决定处理方式。

## 第一步：测试搬瓦工IP是否正常

在测端口之前，先确认IP的基本状态。有两种方式：

**方法一：搬瓦工KiwiVM官方检测**

这是最直接、最权威的方式。搬瓦工官方在KiwiVM后台内置了IP封禁检测功能。

1. 登录你的搬瓦工后台，进入对应VPS的KiwiVM Control Panel
2. 在浏览器地址栏，把URL最后的`main.php`替换成`main-exec.php?mode=blacklistcheck`
3. 页面加载后点击"Test Main IP"

结果很简单：显示`IP NOT BLOCKED`就是正常，显示`IP BLOCKED`就是被封了。

注意一定要先登录KiwiVM再打开检测链接，否则会提示session超时。

**方法二：ping.pe在线测试**

ping.pe 是搬瓦工官方推出的一个网络检测工具，可以同时从全球30多个节点测试你的IP连通性，结果分国内和国外两部分显示。

- 境外绿色 + 国内红色 = IP被封，需要换IP
- 全部绿色 = IP没问题，问题可能出在端口
- 全部红色/超时 = VPS可能关机，或者防火墙屏蔽了ICMP，先检查VPS状态

## 第二步：搬瓦工端口测试（核心步骤）

IP检测通过之后，如果还是连不上，就要测端口了。

### 用 port.ping.pe 测试端口

搬瓦工官方针对端口检测推出了 port.ping.pe，这是目前最准的测试方式之一，国内节点覆盖也比较全。

操作步骤：

1. 打开 port.ping.pe
2. 在输入框中输入：你的IP地址+冒号+端口号，格式如 `1.2.3.4:22` 或者 `1.2.3.4:8765`（SSH端口在KiwiVM后台可以查到）
3. 点击 Go，等待测试完成

结果解读：

- **境外绿色，国内红色**：端口被封了。这个情况不需要换IP，换个SSH端口就能恢复连接
- **全部绿色**：端口正常。问题不在端口，检查一下VPS是否正常运行、防火墙设置是否有问题
- **全部红色**：端口没有开放，检查服务是否正在运行，或者防火墙是否放行了这个端口

### 为什么"能ping通但SSH连不上"？

这是很多人困惑的点。简单说：ping用的是ICMP协议，SSH用的是TCP协议，两者走的不是同一套规则。所以封端口的时候，可以只封TCP/22，ICMP完全不动。结果就是ping有响应，但SSH就是进不去。

这种情况实际上是个好消息——IP本身没问题，换个端口就行，不用花钱换IP。

## 第三步：根据测试结果处理

| 测试结果 | 判断 | 解决方法 |
|---|---|---|
| IP被封（KiwiVM显示BLOCKED） | IP被封 | 付费换IP（$8.79），或迁移机房 |
| IP正常，端口境外通国内封 | SSH端口被封 | 修改SSH端口，换一个没被封的端口 |
| IP正常，端口全部不通 | 服务未启动/防火墙 | 检查VPS状态，放行对应端口 |
| IP正常，端口全部正常 | 网络/配置问题 | 检查本地网络、SSH客户端配置 |

### 情况一：端口被封 → 修改SSH端口

在KiwiVM面板里，可以通过控制台（Console）登录VPS（即使SSH连不上，控制台依然能用），然后修改`/etc/ssh/sshd_config`里的端口配置，换一个10000以上的端口，重启SSH服务即可。改完之后用 port.ping.pe 再测一遍新端口确认正常。

### 情况二：IP被封 → 付费换IP或迁移机房

两种选择：

**付费换IP**：登录搬瓦工后台，找到对应VPS，点击"Request IP Change"，付费 $8.79。官方承诺24小时内完成，实际上大多数情况半小时内就自动换好了，会有邮件通知。

**迁移机房（免费）**：如果你的套餐支持迁移，这是更划算的选择。KiwiVM面板里找到"Datacenter Migration"，换个机房，IP地址自然就变了，完全免费。迁移一般几分钟内完成，数据不会丢失。如果预算允许，迁机房优先考虑——省了$8.79，顺便还可能换到更好的线路。

说实话，碰到IP被封先不要慌，先看看自己的套餐能不能迁机房，能迁就先迁，比付钱换IP划算多了。

## FAQ：搬瓦工端口测试常见问题

**Q：怎么查看我的搬瓦工SSH端口是多少？**

A：登录KiwiVM后台，首页就能看到SSH端口号（标注为"SSH Port"）。搬瓦工默认不是22端口，而是一个随机分配的五位数端口，比如28123这种。SSH客户端连接时记得填对端口号。

**Q：ping.pe测出来国内全红，一定是被封了吗？**

A：不一定。如果VPS处于关机状态，或者系统防火墙屏蔽了ICMP请求，ping也会全红。先确认VPS是开机状态，再结合KiwiVM官方检测工具的结果一起判断，两个都指向"被封"才算确认。

**Q：端口被封了，不换IP能解决吗？**

A：可以。端口被封不等于IP被封，换一个SSH端口（比如从22换到45678）就能继续用。先通过KiwiVM的控制台（Console）登录进去，改SSH配置，换端口，重启SSH服务，完事。

**Q：搬瓦工现在还有免费换IP吗？**

A：没有了。免费换IP服务在2019年已经下线，目前只能付费换，每次$8.79。如果你的套餐支持迁移机房，迁机房是免费的，IP自然就换了，优先考虑这条路。

**Q：搬瓦工连不上，是不是应该先提工单？**

A：一般来说自己先排查，大多数情况能解决。工单响应有时候要等几个小时，自己按上面的步骤查一遍——先测IP，再测端口，确认问题之后按对应方案处理。真的查不出原因再提工单，描述清楚测试结果，效率更高。
